## 5강 비트코인 블록체인 개요

### 블록체인의 역활

- 모든 거래 정보, 발행된 모든 비트코인 현황을 기록한 장부
- 비트코인의 P2P 거래를 가능하게하는 원천



### 블록체인의 목표

- 완전성
  - 모든 거래 내역, 모든 비트코인 현황을 일관성 있게 기록 
- 안전성
  - 비정상 거래 기록 불가
  - 승인된 거래 변경/교체 불가
  - 장부(블록체인) 파괴 불가
- 분산성 / 비집중성
  - 누구나 블록체인 생성에 참여가능
  - 그러나 일관성은 보장되어야함
  - 탈중앙화된 시스템
- 투명성
  - 누구나 블록체인 접근, 검증 및 사용 가능
  - 블록체인을 유지하고 있지 않은 참여자도 블록체인 접근 가능



### 비트코인 네트워크 참여자

- 풀 노트 참여자
- 채굴 노드 참여자
- SPV 노드



### 비트코인 블록

- 채굴자가 일정 시간 동안 비트 코인 네트워크를 통해 교환되는 모든 거래를 검증한 후 모아서 만든 자료구조(장부)
- 생성된 블록은 모든 사용자에게 전달
- 블록체인을 유지하는 참여자는 자신의 블록체인에 수신된 블록을 연결

- 블록생성 간격은 약 10분
  - 10분 동안 발행된 전세계 비트코인 거래를 블록에 기록
- 블록의 크기 : 1 Mbyte로 제한
  - 채굴자간에 블록 전달 지연시간이 커지는 문제를 방지
  - 10분당 승인할 수 있는 거래 수에 제약(초당 7개)
- 블록 생성자는 작업 증명에 가장 먼저 성공한 채굴자가 된다
- **채굴자의 작업 증명 성공 확률** = 채굴자의 해시 파워 / 전체 채굴자의 해시 파워의 총합
- 태초 블록 부터 현재 블록 까지 해시 값으로 유니크하게 연결 되어 있음
  - 블록 조작을 위해서는 뒤에 연결된 모든 블록을 조작해야 가능함
  - 이론적으로는 불가능하나 만약 해시 파워가 절반(50%)이상이면 블록 조작이 가능



### SPV 노드

- 블록 헤더 정보만 유지
- 블록 헤더 : 해시 포인터, 난이도 목표, 넌스, 머클 루트
- 머클 루트 : 블록의 모든 거래에 대한 지문 정보 요약
  - 모든 거래 정보를 저장하고 있다면 생기는 부하를 해결
- 머클 경로 : 특정 거래의 존재 증명에 필요한 일련의 해시 노드
  - 머클 경로의 크기는 log<sub>2</sub>N, N은 전체 거래 수
